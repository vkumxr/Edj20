# Edj20 – Industrial IoT Edge Device

An Industrial IoT (IIoT) edge device developed for smart factory and industrial automation environments.  
The device enables real-time monitoring, control, and communication with industrial sensors and machines using digital, analog, and serial interfaces. :contentReference[oaicite:0]{index=0}

---

# Overview

The **Edj20** is built using the **ESP32-S3** SoC and developed with the **ESP-IDF v5.1.2** framework.  
It is designed to interface with industrial systems and communicate with dashboards or host systems through a UART-based debug interface. :contentReference[oaicite:1]{index=1}

---

# Features

- 4 Digital Inputs (24V)
- 2 Digital Outputs
- 2 Analog Inputs
- Modbus Communication Support
- Current Transformer (CT) Monitoring
- UART Debug Interface
- Configurable Status LEDs
- Real-time Dashboard Integration
- Industrial Automation Ready

# Digital Inputs

- Input Channels: `DI1`, `DI2`, `DI3`, `DI4`
- Voltage Range: `0V – 24V`
- Signal Type: `Digital HIGH/LOW`

---

# Digital Outputs

- Output Channels: `DO1`, `DO2`
- Output Voltage: `24V`
- Signal Type: `Digital HIGH/LOW`

> Note: Digital outputs are independent outputs generated by the Edj20 device. :contentReference[oaicite:3]{index=3}

---

# Analog Inputs

- Input Channels: `AI1`, `AI2`
- Input Range: `0 – 3.3V DC`
- Used for analog voltage sensing and monitoring.

---

# Modbus Interface

The device supports Modbus communication using:

- `A+` → Positive Differential Signal
- `B-` → Negative Differential Signal

Protocol Supported:

- Modbus RTU

---

# Current Transformers (CT)

The Edj20 supports 3 Current Transformers for non-invasive AC current measurement.

Applications include:

- Energy Monitoring
- Machine Current Monitoring
- Industrial Load Analysis

# Test Dashboard

The test dashboard is used to validate all Edj20 interfaces before deployment.

---

# Dashboard Test Coverage

| Test Item | Validation Method |
|---|---|
| Digital Inputs | Apply 24V signal and verify HIGH/LOW state |
| Digital Outputs | Trigger output and verify physical state |
| Analog Inputs | Apply known voltage and verify ADC reading |
| Modbus | Send Modbus RTU frame and verify response |
| LEDs | Verify ON/OFF/BLINK operation |
| Power LED | Visual power confirmation |

---

# Test Flow

1. Dashboard connects to Edj20 via AP+SDA.
2. User initiates testing.
3. LED2 turns ON during testing.
4. Dashboard sequentially validates:
   - Digital Inputs
   - Digital Outputs
   - Analog Inputs
   - Modbus Communication
5. Results are displayed in real-time.
6. LED2 turns OFF after completion.
7. Dashboard generates pass/fail reports.
